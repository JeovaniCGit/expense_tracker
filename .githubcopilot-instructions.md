# Commit Message Conventions

## Purpose

This document defines the commit message format to ensure consistency, readability, and a meaningful project history.

All commit messages **must follow this structure**.

---

## Format

```
<type>(<scope>): <short summary>

<optional detailed description>

<optional footer>
```

---

## 1. Type (Required)

Must be one of the following:

* **feat** – New feature
* **fix** – Bug fix
* **refactor** – Code change that neither fixes a bug nor adds a feature
* **perf** – Performance improvement
* **docs** – Documentation only changes
* **style** – Formatting, missing semicolons, etc (no logic change)
* **test** – Adding or updating tests
* **build** – Changes affecting build system or dependencies
* **ci** – CI/CD configuration changes
* **chore** – Maintenance tasks
* **revert** – Reverts a previous commit

---

## 2. Scope (Optional but Recommended)

Describes the part of the codebase affected.

Examples:

* api
* auth
* ui
* database
* config
* core
* shared

Example:

```
feat(auth): add JWT refresh token support
```

---

## 3. Summary (Required)

* Use **imperative mood** (e.g. “add”, “fix”, not “added” or “fixes”)
* Keep it **concise** (max ~72 characters)
* Do **not** end with a period

Good:

```
fix(api): handle null response from payment provider
```

Bad:

```
fixed stuff
```

---

## 4. Description (Optional but Strongly Encouraged)

Explain **why** the change was made and **what it does**.

* Use bullet points if helpful
* Focus on intent and impact

Example:

```
fix(api): handle null response from payment provider

- Prevents runtime exception when provider returns empty payload
- Adds fallback handling and logging
- Improves resilience of payment flow
```

---

## 5. Footer (Optional)

Used for breaking changes and issue references.

### Breaking Change:

```
BREAKING CHANGE: removes legacy authentication flow
```

### Issue Reference:

```
Closes #123
Fixes #456
```

---

## Examples

### Feature

```
feat(ui): add user profile dropdown menu

- Displays user info and logout option
- Improves navigation accessibility
```

### Bug Fix

```
fix(auth): prevent token expiration race condition

- Synchronizes token refresh logic
- Avoids multiple concurrent refresh calls
```

### Refactor

```
refactor(core): simplify dependency injection setup

- Removes redundant service registrations
- Improves startup performance
```

### Docs

```
docs(readme): update setup instructions
```

### Breaking Change

```
feat(api): redesign authentication endpoints

BREAKING CHANGE: replaces /login with /auth/token
```

---

## Rules for AI (Copilot) Commit Generation

When generating commit messages:

1. Always follow the defined format
2. Infer the correct type based on the changes
3. Include a scope when identifiable
4. Summarize clearly and concisely
5. Add a description for non-trivial changes
6. Avoid generic messages like:

   * "update code"
   * "fix issue"
   * "changes"
7. Do not be verbose without adding value
8. Prefer clarity over cleverness

---

## Anti-Patterns (Do NOT use)

```
misc changes
stuff
fix things
update
WIP
```
